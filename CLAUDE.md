# CLAUDE.md Template

This template should be used whenever creating a new folder in a project to ensure consistent documentation.

```markdown
# [Folder Name]

## Purpose
Brief description of what this folder is for and its role in the project.

## Contents
- **Key files and their functions**
- **Subdirectories and their purposes**
- **Configuration files and their roles**

## Dependencies
- Libraries, frameworks, or other folders this depends on
- External services or APIs used
- Environment variables or configuration required

## Development Notes
- Setup instructions specific to this folder
- Common commands or scripts to run
- Known issues or gotchas
- Testing approach for this component

## Relationships
- How this folder connects to other parts of the project
- Data flow in and out of this component
- Integration points with other services

## Last Updated
[Current Date] - [Brief description of last major change]

---
*This file was automatically generated by Claude CLI*
*Update this file when making significant changes to the folder structure or purpose*
```

## Global Rule Implementation

When Claude creates any new folder in a project, it should:

1. **Always create a CLAUDE.md file** in the new folder
2. **Use this template** as the base structure
3. **Fill in relevant information** based on the folder's purpose
4. **Update existing CLAUDE.md files** when folder contents change significantly
5. **Include contextual information** specific to the project and folder role

## Template Customization

The template should be adapted based on folder type:

- **Component folders**: Focus on props, state, and UI patterns
- **Service folders**: Emphasize API endpoints and data handling
- **Config folders**: Detail configuration options and environment setup
- **Test folders**: Describe testing strategy and coverage
- **Documentation folders**: Outline documentation structure and guidelines

## Maintenance

- Review CLAUDE.md files during code reviews
- Update when adding new major features or dependencies
- Keep the "Last Updated" section current
- Ensure consistency across the project
